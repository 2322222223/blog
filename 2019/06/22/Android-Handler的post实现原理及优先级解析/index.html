Handler的post(Runnable)实现原理及优先级解析

1. 简介

在Android中，Handler类用于管理线程之间的通信，主要通过消息（Message）和可运行任务（Runnable）来实现。本文将详细介绍Handler的post(Runnable)方法的实现原理，以及callback、runnable和msg的执行优先级，并结合源码和注释进行深入解析。

2. Handler的post(Runnable)实现原理

Handler的post(Runnable)方法用于将一个Runnable任务添加到消息队列中，并在指定的时间执行。其实现原理是将Runnable封装成一个Message，然后将该Message添加到消息队列中。

2.2 sendMessageDelayed的实现

sendMessageDelayed方法最终调用sendMessageAtTime(Message, long)方法，将消息添加到指定时间点的消息队列中。

2.3 sendMessageAtTime的实现

在sendMessageAtTime方法中，首先获取Handler所关联的消息队列mQueue，然后调用enqueueMessage(MessageQueue, Message, long)方法将消息添加到消息队列中。

2.4 MessageQueue.enqueueMessage的实现

enqueueMessage方法将消息按时间顺序插入到消息队列中。

3. callback、runnable和msg的执行优先级

在Handler的消息处理机制中，callback、runnable和普通的msg有不同的执行优先级。

3.1 callback的执行优先级

当Message的callback属性不为空时，Handler会优先执行callback。

在dispatchMessage方法中，如果Message的callback属性不为空，则调用handleCallback(Message)方法执行callback。handleCallback方法调用Runnable的run方法。

3.2 runnable的执行优先级

通过post(Runnable)方法添加到消息队列的Runnable任务会被封装成Message，其callback属性指向该Runnable，因此其执行优先级与callback相同。

3.3 msg的执行优先级

普通的消息msg没有callback属性，Handler会调用其handleMessage(Message)方法处理消息。

当callback属性为空时，Handler会先检查自身的mCallback是否为空，如果不为空，则调用mCallback的handleMessage(Message)方法。如果mCallback也为空，则调用Handler的handleMessage(Message)方法处理消息。
